

The game will be documented in the following way.

1. docs/REQUIREMENTS.md will contain details about all the required features and gemaplay behaviour of the game.

2. docs/DESIGN.md will contain details about how we plan to implement the requirements in the codebase.

3. docs/TESTING.md will describe how the game will be teested.

4. docs/ARCHITECTURE.md will contain details about how the game modules are structured and implemented, and how the different modules interact.

5. docs/TODOS.md will contain current tasks we are working on.

6. docs/PATHS.md will contain an outline of the folder structure to make it easier for AI to navigate the project.

7. docs/AI.md will contain LLM-specific instructions to optimise it's accuracy, performance, cost, and behaviour.

The documentation should be kept up to date as we build the game.
